# ООП: Лабораторная работа №5

## Тема: Моделирование работы лифта с использованием Qt Signals/Slots

Разработать приложение на C++ с использованием фреймворка Qt, моделирующее работу пассажирского лифта в пятиэтажном здании. Запрещается использовать циклы ожидания и блокировки потока интерфейса.

\---

## 1\. Архитектура приложения

Программа должна быть реализована с использованием объектно-ориентированного подхода и включать следующие основные сущности:

* контроллер лифта;
* кабина лифта;
* двери лифта;
* графический интерфейс.

Взаимодействие между объектами должно выполняться через механизм qt - сигналы и слоты.

\---

## 2\. Функциональность лифта

Лифт должен:

* принимать вызовы с этажей;
* принимать команды из кабины;
* перемещаться между этажами;
* открывать и закрывать двери;
* обрабатывать несколько заявок;
* определять направление движения;
* останавливаться на промежуточных этажах при наличии вызова по пути следования.

\---

## 3\. Возможные состояния

### Кабина

* движение;
* остановка;
* подготовка к движению;
* заблокирована;
* разблокирована.

### Двери

* открываются;
* открыты;
* закрываются;
* закрыты.

### Контроллер

* свободен;
* движение;
* обновление цели;
* достижение целевого этажа.

\---

## 4\. Имитация времени

Работа лифта должна моделироваться с помощью `QTimer` таймеров:

* время перемещения между этажами;
* время открытия дверей;
* время закрытия дверей;
* время ожидания на этаже.

\---

## 5\. Графический интерфейс

Интерфейс должен содержать:

* кнопки вызова лифта на этажах;
* кнопки выбора этажей внутри кабины.

Минимальное оличество этажей — 5.

\---

## 6\. Логирование

В консоль минимально должны выводиться:

* вызов лифта;
* начало движения;
* прибытие на этаж;
* открытие/закрытие дверей;
* выбор новой цели;
* переходы состояний.
